# Week 9 Team Role Split — 라이프사이클 × 기술 영역 매트릭스

> 4명 팀 기준 sample. 팀 사이즈가 다르면 한 사람이 2 영역을 겸할 수 있음.

## 2축 분담 원칙

- **세로 축**: 기술 영역 (인증·조회·동시성·캐시 등) — 코드 PR 단위
- **가로 축**: 라이프사이클 단계 (기획·코딩·테스트·리뷰·운영) — 팀 산출물 단위

각 팀원은 본인 기술 영역의 **코딩**을 책임지고, **라이프사이클 한 단계의 팀 산출물**을 추가로 책임집니다.

## 매트릭스 sample (4명 팀)

| 팀원 | 기술 영역 (코딩 PR) | 라이프사이클 책임 단계 | PR에 남길 것 |
| --- | --- | --- | --- |
| 팀원 A | 인증/권한 (로그인, 토큰, 인가 예외) | **기획** — `PRD.md` + Jira MCP 티켓 분해 | 보안 예외 근거 + PRD에 본인 기여 |
| 팀원 B | 핵심 조회 (검색, 인덱스, 응답 DTO) | **테스트** — `tests/e2e/` Playwright MCP 시나리오 | 인덱스/응답 시간 근거 + E2E 시나리오 1개 |
| 팀원 C | 상태 전이/동시성 (트랜잭션, 락, 재시도) | **리뷰** — `.github/workflows/ai-review.yml` (Claude Actions/CodeRabbit) | 경쟁 상태 재현/해결 + AI 리뷰 도입 결과 |
| 팀원 D | 캐시/비동기 (Redis, 이벤트, AI 보조 기능) | **운영** — `MONITORING.md` (Sentry MCP) | 캐시 전략 + 모니터링 설정 |

## 코딩 단계 (책 7주차 04~07)는 모두 함께

`CLAUDE.md`(팀 헌법)와 `API-CONTRACT.md`는 4명이 같이 작성. 본인 기술 영역에 해당하는 부분 본인이 채움.

## 팀 사이즈가 다를 때

### 3명 팀

- 팀원 A: 인증/권한 + **기획**
- 팀원 B: 핵심 조회 + 동시성 + **테스트**
- 팀원 C: 캐시/비동기 + 리뷰 + **운영** (리뷰는 도구 설정만, 깊은 검토는 팀 전원)

### 5명 팀

- A, B, C, D는 위와 동일
- 팀원 E: AI 보조 기능 단독 담당 + **협업 가속** (gh CLI / Sub-agents / claude-squad 도입)

## PR 본문 명시 형식

```
- 기술 영역: 핵심 조회
- 라이프사이클 단계: 코딩 + 테스트
- 본 PR이 다룬 산출물: ProductSearchService, ProductRepository 인덱스, tests/e2e/search-flow.spec.ts
```

## 핵심

학생 개인 입장에서 "내가 어떤 기술 영역을 맡았고, 어떤 라이프사이클 단계를 책임졌고, 어떻게 검증했는지"가 PR 하나로 설명 가능해야 합니다. Week 10 면접에서 이 매트릭스가 그대로 답변 자료가 됩니다.
